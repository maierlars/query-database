# Generation of deterministic datasets

For reliable and reproducible performance tests, it is important to generate deterministic datasets. 
Slight drifts in data distribution or data layout can lead to significant performance variance.

This document contains a checklist for generating deterministic datasets.

- **Seeded randomness**: If you employ a random number generator to generate the dataset, make sure that it is seeded properly and always use the same seed.
- **Data Distribution**: Make sure that all shard keys (usually the primary key) are specified. This is two fold: since the primary key is usually used for sharding but is also usually autogenerated, the data distribution is random. Make sure that always the same documents end up in the same shard.
